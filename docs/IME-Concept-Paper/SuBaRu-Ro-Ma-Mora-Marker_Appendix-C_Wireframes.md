# 付録C：画面イメージの説明

本付録では、すばる式ローマ字を用いた日本語IME入力支援の画面イメージを、テキストベースのワイヤーフレームとして示す。[file:1] 付録Bの入力フローと対応づけながら、高速ユーザー向け、初心者・低速ユーザー向け、英字入力からの救済的切替の3場面を整理する。[file:1]

---

## C-1. 高速ユーザー向け画面イメージ

図C-1は、高速にローマ字入力を行う利用者向けの画面イメージである。[file:1] 入力直後に、ごく短時間だけすばる式ローマ字の大文字補完表示が現れ、その後すみやかに変換処理へ接続する構成を想定する。[file:1]

```text
┌─────────────────────────────────────┐
│ アプリケーションウィンドウ（例：テキストエディタ）         │
│ ─────────────────────────────── │
│  ……既存のテキスト……                                    │
│                                                         │
│  [入力中カーソル位置]                                   │
│   TopPu                                                 │
│      ↑                                                  │
│   （すばる式ローマ字による大文字補完表示：短時間フラッシュ）│
│                                                         │
└─────────────────────────────────────┘
```

この例では、利用者が `toppu` と入力した際に、IME上では `TopPu` として拍境界が視覚化される。[file:1] 促音の位置を子音小文字で残し、その直後の拍頭子音を大文字化することで、すばる式ローマ字らしい視覚的な引っかかりが生じる。[file:4]

---

## C-2. 初心者・低速ユーザー向け画面イメージ

図C-2は、タイピングに不慣れな利用者に対して、入力したローマ字列とかな表記との対応を段階的に示す画面イメージである。[file:1] 本構想書では、大文字補完表示がまず現れ、その後にかな表示へ滑らかに切り替わる流れを想定している。[file:1]

```text
ステップ1：ローマ字入力直後
┌─────────────────────────────────────┐
│ ……既存のテキスト……                                    │
│                                                         │
│  GakkoU                                                 │
│   ↑  ↑                                                  │
│   │  └─ 後続拍の開始位置                               │
│   └──── 語頭拍の開始位置                               │
│                                                         │
└─────────────────────────────────────┘

ステップ2：かな表示へのフェイド切替中
┌─────────────────────────────────────┐
│ ……既存のテキスト……                                    │
│                                                         │
│  GakkoU   →   がっこう                                  │
│   │             │                                       │
│   │             └─ 対応するかな表記                    │
│   └─ 入力したローマ字列                                │
│                                                         │
└─────────────────────────────────────┘

ステップ3：かな表示のみ
┌─────────────────────────────────────┐
│ ……既存のテキスト……                                    │
│                                                         │
│  がっこう                                                │
│                                                         │
└─────────────────────────────────────┘
```

この例では、`gakkou` の入力に対して `GakkoU` を提示し、語頭と後続モーラの開始位置を大文字で示している。[file:4] その後にかな表示へ切り替えることで、利用者はローマ字入力とかな表記との対応関係を段階的に理解しやすくなることが期待される。[file:1]

## C-3. 英字入力からの救済的切替画面イメージ

図C-3は、英字直接入力モードで誤って入力した文字列を、日本語IMEローマ字入力モードへ切り替えた際に救済する場面の画面イメージである。[file:1] 構想書では、入力済み英字列をローマ字列として再評価し、大文字補完表示を伴ってかな変換候補へ接続する経路を想定している。[file:1]

```text
ステップ1：英字直接入力モードでの誤入力
┌─────────────────────────────────────┐
│ 入力モード：英字直接入力                                │
│ ……既存のテキスト……                                    │
│                                                         │
│  kitte                                                  │
│      ↑                                                  │
│  （利用者は日本語入力のつもりで打鍵している）            │
└─────────────────────────────────────┘

ステップ2：日本語IMEローマ字入力モードへ切替
┌─────────────────────────────────────┐
│ 入力モード：日本語IMEローマ字入力                        │
│ ……既存のテキスト……                                    │
│                                                         │
│  KitTe                                                  │
│   ↑ ↑                                                   │
│   │ └─ 後続拍の開始位置                                │
│   └── 語頭拍の開始位置                                 │
│                                                         │
└─────────────────────────────────────┘

ステップ3：かな変換候補として提示
┌─────────────────────────────────────┐
│ 入力モード：日本語IMEローマ字入力                        │
│ ……既存のテキスト……                                    │
│                                                         │
│  KitTe  →  きって  （変換候補リストの一部として表示）     │
│                                                         │
└─────────────────────────────────────┘
```

この例では、英字列 `kitte` が日本語IME側で再解釈され、`KitTe` というすばる式ローマ字表示に組み替えられる。[file:1][file:4] その結果、利用者は誤入力を打ち直さずに、かな変換候補として処理が継続されたことを視覚的に把握できる。[file:1]
