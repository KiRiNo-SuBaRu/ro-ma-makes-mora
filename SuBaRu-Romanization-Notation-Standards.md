
# すばる式ローマ字表記法 規格（Part-002 完整稿）

本書は、訓令式ローマ字を基底入力として、仮名塊を視覚的に立ち上げる「すばる式ローマ字」の筆記法を定義するための草案である。
目的は、一般的な欧文単語と日本語ローマ字列とを初見の字面で見分けやすくし、仮名単位・仮名塊単位を強く意識させることにある。

## 1. 目的と適用範囲

すばる式ローマ字は、学校教育用の厳密な正書法そのものを置き換える規格ではなく、訓令式ローマ字の入力列をもとに、視覚的な違和感を利用して日本語の音節性を強調する筆記法として設計される。
そのため本規格は、音声転写の忠実さだけでなく、「どこをひと塊の仮名として見せるか」という表記設計を中核に置く。

適用対象は、半角小文字の訓令式ローマ字入力を基本とする日本語表記である。
公開表示では、基本変換による表示と、後段の表示例外とを分離して扱う構成を採る。

## 2. 基本原則

### 2.1 子音先頭の立ち上げ

子音を表す文字で始まる仮名単位は、先頭字のみを大文字化する。
したがって、通常の `CV` や `CyV` は `Ka`、`Si`、`Gyo` のように表記する。

### 2.2 独立母音の立ち上げ

子音を伴わない独立したア行母音は、大文字で表記する。
たとえば、再解釈上で独立母音と判定された `o` や `e` は `O`、`E` のように示す。

### 2.3 撥音 `ん`

撥音は小文字 `n` を基本とし、独立した大文字化対象とはしない。
直後が母音字または `y` の場合のみ `n'` とし、それ以外では `n` を用いる。

### 2.4 促音 `っ`

促音は入力解析上では独立トークンとして保持するが、公開表示では単独文字としては立てず、後続子音を肥大化させる要素として扱う。
そのため `Q + CV` は `ppu`、`tta` のような重子音入りの塊として再構成する。

## 3. 入力前提

入力は半角小文字の訓令式ローマ字とする。
空白は語境界として保持し、変換処理はまず語または表記単位の分割から始める。

句読点、中点、括弧、ハイフンを含む文字列は、必要に応じて単なる語ではなく「表記単位」として扱う。
これは、複合語や演出的なまとまりを、空白区切りより上位の単位として保持するためである。

### 3.1 分かち書きの推奨

すばる式ローマ字では、可読性と仮名塊の視認性を高めるため、語または表記単位ごとの分かち書きを推奨する。
とくに学習用・観察用・公開用の表示では、連続したローマ字列を詰めて書くよりも、意味上または表記上のまとまりごとに空白で区切る方が、仮名塊の境界と違和感の位置とを把握しやすい。

分かち書きは必須ではないが、本規格の運用上は推奨事項として扱う。
ただし、中点・句読点・括弧・ハイフンを含む複合語については、必要に応じて空白で分断せず、ひとまとまりの表記単位として保持してよい。

とくに、長音、促音、助詞は、分かち書きした方が視認性と教育効果の両面で有利であるため、優先的に区切ることを推奨する。
長音は仮名塊の持続感を、促音は子音肥大化の位置を、助詞は文法上の切れ目を、それぞれ空白によって見失いにくくできるためである。

たとえば、長音を含む語、促音を含む語、格助詞や係助詞を伴う連続列では、字面を詰めるよりも、`KaHanSuu No`、`To ppu No`、`Ni HonGo No` のように観察単位を明示する書き分けが推奨される。

## 4. 処理単位

本規格では、入力文字列を次の順で処理する。

1. 表記単位へ分割する。
2. 表記単位をモーラ風トークン列へ分解する。
3. トークン列を仮名塊へ再束ねする。
4. 基本表示を生成する。
5. 必要に応じて表示例外を適用する。

モーラ風トークンは、少なくとも `CV`、`CyV`、`N`、`Q`、`V` を基本分類とする。
ここで重要なのは、最終表示がトークン列の単純な逐次大文字化ではなく、再束ね後の仮名塊に基づいて決まる点である。

### 4.1 モーラ風トークン定義（要約）

- `V` : 子音を伴わない単独母音（例: `a`, `i`, `u`, `e`, `o`）。
- `CV` : 子音+母音から成る基本単位（例: `ka`, `si`, `te`, `no`）。
- `CyV` : 子音+`y`+母音の拗音系単位（例: `kyo`, `gyo`, `sya`, `tyu`）。
- `N` : 撥音単位。直後が子音または語末のときに抽出される（`ken` 中の `n` など）。
- `Q` : 促音単位。後続 `CV` / `CyV` の重子音を予告する。
- `C?` : 上記に収まらない孤立子音などの暫定処理用。

## 5. 仮名塊再解釈

### 5.1 `CVn` / `Vn`

`CV + N + 子音` および `CV + N + 語末` は、`CVn` の一塊として再束ねする。
同様に、`V + N + 子音` および `V + N + 語末` は `Vn` の一塊として再束ねする。

この規則により、`ken` は `Ken`、`han` は `Han`、`en` は `En` のように表記される。
語末 `n` は独立クラスタを作らず、直前塊へ小文字従属させる。

ただし、直後が母音字または `y` の場合は `n'` を優先し、`CVn / Vn` への吸収は行わない（例: `sinya` → `Sin'Ya`、`kinyoubi` → `Kin'YoUBi` といった方向性）。

### 5.2 `Q + CV` / `Q + CyV`

促音は単独表示せず、後続塊に吸収する。
そのため `Q + CV` は `ppu`、`tta` のような重子音入り塊へ、`Q + CyV` は `kkyo`、`ttya` のような塊へ再束ねする。

促音吸収によって成立した塊は、その内部構造は固定するが、塊の **直後に続く母音トークン** については、長母音・独立母音化の判定対象とする。
これにより、`kitteiru` → `KitteIRu`、`gakkou` → `GakkoU` のような表示を許容する。

### 5.3 長母音と二重母音

`CV + 同母音V` は、長母音塊として保持する。
たとえば `su + u` は `Suu` のように表記する。

`CV + ai/oi/ui` は、原則として二重母音塊として保持する（ただし後述の `ai` 層別規則との整合は別表B参照）。

### 5.4 独立母音化

`CV + V` または `CyV + V` の連接は、常に連母音として扱うのではなく、独立母音化する場合と連母音塊として維持する場合を分ける。
たとえば `siteori` は `SiTeORi`、`osaete` は `OSaETe` のように、`eo` や `ae` を分割して独立母音を立ち上げる。

長母音・二重母音としての既定扱いがある `ou / ai / ei` については、位置別の既定表示を別表Bで定める。

### 5.5 連母音維持

一部の母音連接は、独立母音へ分離せず塊として維持する。
たとえば `sieawo` は `SieaWo` のように、`siea` を一続きの塊として観察対象にできる。

連母音維持は一般規則ではなく、観察・演出上の必要に応じて指定される特例として扱う（詳細は別表B）。

## 別表A. 再束ね優先順位表

本別表は、複数の再束ね候補（促音、撥音、長母音、独立母音化など）が競合する場合に、どの規則を優先するかを示す。

| 優先度 | 規則名 | 形式 | 代表例 | 優先理由 |
| :-- | :-- | :-- | :-- | :-- |
| 1 | 促音吸収 | `Q + CV` / `Q + CyV` | `toppu`, `kitte`, `gakkou` | 促音は単独表示せず、後続子音の性質を先に確定する必要がある。 |
| 2 | 撥音処理 | `CV + N` / `V + N`（ただし母音・`y` 前は `n'`） | `ken`, `enzin`, `sinya`, `kinyoubi` | `n` の従属先を早めに決め、`n'` 条件との衝突を先に解消する。 |
| 3 | 特定母音連接の既定値 | `ou / ai / ei` | `gakkou`, `gyoukai`, `kaihatu`, `seisyohou`, `yasai` | すばる式の中核的な見え方を作る既定層であり、一般独立母音化より先に判定する。 |
| 4 | 一般独立母音化 | その他の `CV + V` / `CyV + V` | `siteori`, `osaete`, `kitteiru` | 既定値の対象外に対し、局所的な視覚的切れ目を立てる。 |
| 5 | 連母音維持 | 特例的に維持する母音列 | `sieawo` | 分離しない方が観察目的に合うものを限定的に維持する。 |
| 6 | 単純塊保持 | 残余の `CV` / `CyV` / `V` | `ka`, `si`, `gyo`, `a` | 上記規則に当たらない要素を通常の仮名塊として確定する。 |

## 別表B. 独立母音化／連母音維持 判定表

本別表は、母音連接 `CV + V` / `CyV + V` に対して、どの組み合わせを既定で独立母音化し、どれを連母音として維持するかを示す。

### B.1 特定母音連接（ou / ai / ei / oi / ui）

| 型・位置 | 既定処理 | 代表例 → すばる式 | 説明 |
| :-- | :-- | :-- | :-- |
| `ou` 語頭 | 分割寄り・独立化候補 | `ouji` → `OUZi` | 語頭では独立母音の立ち上がりを優先して分割を許容する。 |
| `ou` 語中 | 維持寄りだが分割許容 | `toukyou` → `TouKyoU` | 可読性を優先して長音寄りに扱うが、演出上は `ToU` などへの分割も許容する。 |
| `ou` 語末 | 独立母音化優先 | `gakkou` → `GakkoU` | 語末長音は `...oU` の形で末尾母音を立てるのを既定とする。 |
| `ai` 語頭 | 分割寄り | `aiko` → `AiKo` / `AIKo` | 固有名詞的な揺れを許しつつ、`Ai` をひと塊として強調する方向を既定とする。 |
| `ai` 語中 | 独立母音化優先 | `kaihatu` → `KaIHaTu` | 語中 `ai` は視覚的切れ目を立てるコア領域として `aI` を既定とする。 |
| `ai` 語末 | 独立母音化優先 | `yasai` → `YaSaI` | 語末 `ai` も `...aI` を既定表示とする。 |
| `ei` 語頭 | 独立母音化優先 | `eizi` → `EIZi` | 語頭 `ei` は `EI` として立て、長いエの立ち上がりを強調する。 |
| `ei` 語中 | 独立母音化優先 | `seisyohou` → `SeISyoHoU` | 語中 `ei` は長音化より `eI` を優先する。 |
| `ei` 語末 | 独立母音化優先 | `orei` → `OReI` | 語末 `ei` も `...eI` を既定表示とする。 |
| `oi`（一般二重母音） | 維持寄り | `oi` → `Oi` | 特別扱いはせず、読みやすさを優先して二重母音として一塊で保持する。 |
| `ui`（一般二重母音） | 維持寄り | `sui` → `Sui`（想定） | 出現頻度と違和感のバランスから、当面は一般二重母音として維持寄りに扱う。 |

### B.2 一般独立母音化と連母音維持

| 型・範囲 | 既定処理 | 代表例 → すばる式 | 説明 |
| :-- | :-- | :-- | :-- |
| `eo`（一般例） | 一般独立母音化 | `siteori` → `SiTeORi` | `ou / ai / ei` の特別枠に含まれないため、`Te + O` を分割して独立母音化する。 |
| `ae`（一般例） | 一般独立母音化 | `osaete` → `OSaETe` | `sa + e` を分割し、`E` を立てて視覚的な切れ目を作る。 |
| その他の連母音 | 原則維持または個別判定 | `sieawo` → `SieaWo` | 観察・演出目的で特に維持したい連母音のみ、連母音維持の特例として残す。 |

### B.3 適用順序

1. まず `ou / ai / ei` のいずれかに該当するかを判定し、該当する場合は B.1 の位置別既定値を適用する。
2. `oi / ui` は特別扱いせず、原則として「維持寄りの一般二重母音」として処理する。
3. 上記に該当しない `CV + V` / `CyV + V` について、B.2 に基づき一般独立母音化または連母音維持の特例を適用する。

## 6. 複合語と表記単位

中点・句読点・括弧・ハイフンを含む文字列は、必要に応じて「複合語」として1単位で扱う。
このとき、複合語は空白区切りの語より上位の表記単位として扱われる。

例外照合では、まず元の表記単位そのままで照合し、必要に応じて先頭・末尾の句読点を除いた正規化キーでも照合する。
これにより、前後の語が変わっても、複合語そのものに対して演出的例外を安定して適用できる。

## 別表C. 複合語例外キーの正規化規則

本別表は、複合語や句読点付き表記単位に対して、表示例外辞書で安定して照合するための正規化キー生成規則を示す。

### C.1 正規化の目的

表示例外辞書で、複合語や句読点付き表記単位に安定してマッチさせるため、「見た目が少し変わっても同じ語として扱えるキー」を定義する。

### C.2 正規化手順

複合語に対する例外照合キーは、次の手順で生成する。

1. 原文そのままの表記単位を第1キーとする。
2. 先頭および末尾の句読点・括弧・引用符のみを除去した文字列を第2キーとする。
3. 中点（・）、ハイフン（-）、長音記号（ー）は内部構造を示す記号として保持し、空白には変換しない。
4. 英字については、大文字・小文字の差を照合時に無視し、内部保存は小文字へ正規化する。
5. 連続する空白は単一空白へ正規化する。

表示例外辞書は、基本的に第2キーで管理し、第1キーは「外側に記号が付いたままのケース」へのフォールバック照合に用いる。

### C.3 正規化例

| 入力表記単位 | 第1キー | 第2キー | 説明 |
| :-- | :-- | :-- | :-- |
| `eigo・ratengo` | `eigo・ratengo` | `eigo・ratengo` | 中点は複合語内部構造として保持する。 |
| `「eigo・ratengo」` | `「eigo・ratengo」` | `eigo・ratengo` | 外側の引用符のみを除去し、内部は変更しない。 |
| `(seisyohou)` | `(seisyohou)` | `seisyohou` | 外側の括弧のみを除去し、語本体をキーとする。 |
| `kai-hatu` | `kai-hatu` | `kai-hatu` | ハイフンは複合語境界として保持し、空白化しない。 |
| `EIGO・RATENGO` | `EIGO・RATENGO` | `eigo・ratengo` | 照合時は小文字に正規化し、大文字小文字を区別しない。 |

## 7. 基本表示と表示例外

本規格では、基本表示と表示例外を明確に分離する。
基本表示は再束ね規則に基づく機械生成結果であり、表示例外は公開上の字面調整のための後段ルールである。

表示例外は、`display-exception-rules.json` 相当の辞書で管理する。
この辞書は、語単位にも複合語単位にも適用できる構成が望ましい。

## 別表D. 基本表示と表示例外の責務分担表

本別表は、再束ね規則に基づく「基本表示」と、`display-exception-rules.json` による「表示例外」の責務を区別するための一覧である。

| 項目 | 基本表示で扱う | 表示例外で扱う | 説明 |
| :-- | :-- | :-- | :-- |
| 子音先頭の大文字化 | ○ | - | `CV` / `CyV` の先頭子音を大文字化する規則（例: `ka` → `Ka`）。 |
| 独立母音の大文字化 | ○ | - | 再解釈上の独立母音を `A/I/U/E/O` として立てる規則。 |
| `n / n'` 判定 | ○ | - | 撥音 `n` の基本処理と、直後が母音・`y` の場合の `n'` 判定（例: `Sin'Ya`）。 |
| 促音吸収 | ○ | - | `Q + CV` / `Q + CyV` を重子音塊に束ねる処理（例: `toppu` → `Toppu`）。 |
| `CVn / Vn` 形成 | ○ | - | 撥音を直前塊へ従属させる処理（例: `ken` → `Ken`）。 |
| `ou / ai / ei` の位置別既定値 | ○ | - | 語頭・語中・語末ごとの既定表示（例: `gakkou` → `GakkoU`, `yasai` → `YaSaI`）。 |
| 一般独立母音化 | ○ | - | `eo`, `ae` など特例以外の連接に対する独立母音化（例: `siteori` → `SiTeORi`）。 |
| 連母音維持の特例（規格に明示したもの） | ○ | ○ | 観察目的で維持する連母音（例: `sieawo` → `SieaWo`）の「規格としての核」は基本表示で定義し、語彙ごとの追加は表示例外で拡張する。 |
| 分かち書きの推奨単位 | ○ | - | 仮名塊・助詞・長音・促音を分ける推奨方針（例: `Ni HonGo No`）。 |
| 固有名詞固有の見せ方 | - | ○ | 作品固有名詞としての特別な大文字化や崩し（ロゴ用表記など）。 |
| 複合語単位での強調 | - | ○ | `eigo・ratengo` のような複合語に対する特別な表示（例: `EIGo・RaTenGo`）。 |
| UI 上の演出調整 | - | ○ | 行末のみの強調、特定画面での強調抑制など、表示メディア依存の調整。 |
| 歴史的仮名遣い・語源的綴り | - | ○ | 歴史的・語源的な事情から、通常の規則と異なる表示を維持したい場合。 |

基本方針として、「仮名塊の境界と違和感の位置」が再束ね規則から自動的に再現できる範囲はすべて基本表示側で担保し、それでも足りない作品固有の演出は表示例外側へ委ねる構成とする。

## 9. 規格上の設計原則

すばる式ローマ字では、違和感は誤記ではなく、仮名塊を立たせるための演出である。
したがって、規格設計では「どこに視覚的ひっかかりを置くか」を、音読可能性と同じくらい重要な要件として扱う。

また、規格としては、すべてを語彙例外へ逃がすのではなく、できる限り再束ね規則として一般化し、そのうえでなお必要なものだけを表示例外に委ねるのが望ましい。
この分離があることで、表記法としての一貫性と、創作上の演出自由度とを両立しやすくなる。
